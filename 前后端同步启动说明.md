# LALC 前后端同步启动指南

## 📌 问题诊断

你遇到的两个问题已经被识别和解决：

### 问题 1：程序无法识别并自动操作边狱巴士
**原因分析**：
- ✅ 后端 **可以** 识别游戏窗口（日志证实）
- ❌ 问题是前后端没有同时运行
- ❌ 前端没有正确连接或发送启动请求

**日志证据**：
```
检测到窗口: LimbusCompany.exe - LimbusCompany
位置: (2, 39, 891, 515) (匹配分数: 110)
✓ 自动检测到游戏窗口: left=2, top=39, width=891, height=515
```

### 问题 2：前后端没有同时启动
**原因**：旧脚本只启动后端，不启动前端应用

**解决方案**：使用新的一键启动脚本

## 🚀 快速启动（推荐）

直接双击或运行以下脚本：

```bash
./一键启动.sh
```

这个脚本会：
1. ✓ 清理 macOS 系统文件（.DS_Store）
2. ✓ 检测游戏窗口
3. ✓ 启动后端服务（ws://localhost:8765）
4. ✓ 启动前端应用（macOS 窗口）
5. ✓ 自动保持两者运行

## 🔧 系统权限配置（重要）

如果后端无法识别游戏窗口或无法执行操作，需要授予权限：

### macOS 权限设置
1. **系统设置** → **隐私与安全性**
2. 左侧菜单找到：
   - **屏幕录制** → 添加 `Terminal` 或你使用的终端应用
   - **辅助功能** → 添加 `Terminal` 或你使用的终端应用

3. **重启终端应用**，重新运行启动脚本

### 权限验证
运行以下命令检查权限：
```bash
# 检查屏幕录制权限
tccutil list ScreenCapture2 | grep -i terminal

# 检查辅助功能权限
tccutil list AccessibilityAwareness | grep -i terminal
```

## 🐛 故障排查

### 情况 1：前端应用无法启动
**症状**：一键启动脚本运行后看不到应用窗口

**解决**：
```bash
# 检查编译是否成功
ls -lh /Users/Zhuanz/Desktop/LixAssistantLimbusCompany-master/lalc_frontend/build/macos/Build/Products/Release/lalc_frontend.app

# 如果不存在，需要重新编译
cd lalc_frontend
flutter build macos --release
```

### 情况 2：后端无法启动
**症状**：脚本卡在"等待后端启动"

**检查日志**：
```bash
tail -50 lalc_backend/logs/server.log | grep -i error
```

**常见原因**：
- ❌ `.DS_Store` 文件干扰图像初始化
  ```bash
  find lalc_backend -name ".DS_Store" -delete
  ```
- ❌ 缺少依赖
  ```bash
  source lalc_backend/.venv/bin/activate
  pip install -r lalc_backend/requirements-macos.txt
  ```
- ❌ 权限不足
  ```bash
  chmod -R 755 lalc_backend
  ```

### 情况 3：前端连接不上后端
**症状**：前端显示"无法连接本地服务器"或"已断开连接"

**检查后端**：
```bash
# 验证后端运行
ps aux | grep "main.py" | grep -v grep

# 验证端口监听
lsof -i :8765

# 测试 WebSocket 连接（需要 websocat）
brew install websocat
websocat ws://127.0.0.1:8765
```

**解决**：
1. 关闭前端窗口
2. 等待后端自动超时或手动重启：
   ```bash
   pgrep -f "main.py" | xargs kill -9
   # 然后重新运行一键启动脚本
   ```

### 情况 4：游戏无法被识别或无法操作
**症状**：后端日志中没有"检测到窗口"信息

**检查**：
1. **游戏是否运行？**
   ```bash
   ps aux | grep -i "limbus\|crossover\|wine" | grep -v grep
   ```

2. **终端权限是否足够？**
   - 打开系统设置，检查隐私与安全性中是否授权了屏幕录制和辅助功能

3. **游戏窗口名称是否正确？**
   ```bash
   # 列出所有窗口
   cd lalc_backend && python -c "
   from utils.window_detector import list_all_windows
   list_all_windows()
   "
   ```

4. **重新授权权限**：
   ```bash
   # 移除旧的权限记录
   rm -rf ~/Library/Application\ Support/CrashReporter/DiagnosticMessagesHistory.plist
   
   # 重启 Terminal
   # 再次运行脚本时会重新请求权限
   ```
