## ⚠️ 重要说明

### 1. 平台兼容性

当前打包的依赖为 **Apple Silicon (arm64)** 架构。

**Intel Mac 用户**需要重新下载依赖：

```bash
cd lalc_backend
rm -rf deps_macos
pip download --dest deps_macos -r requirements-macos.txt \
    --platform macosx_10_9_x86_64
```

**通用方案**（同时支持 Intel 和 Apple Silicon）：

```bash
# 下载 universal2 架构
pip download --dest deps_macos -r requirements-macos.txt \
    --platform macosx_10_13_universal2 --no-deps

# 然后下载平台特定的依赖
pip download --dest deps_macos -r requirements-macos.txt
```

### 2. Python 版本

当前依赖为 **Python 3.13** 编译，兼容 3.10-3.13。

如果用户使用其他版本，可以重新下载：

```bash
pip download --dest deps_macos -r requirements-macos.txt \
    --python-version 3.12
```

### 3. 传输大文件

如果通过 Git 托管（GitHub/GitLab），171MB 的依赖包可能遇到限制：

**解决方案 A**: 使用 Git LFS（大文件存储）

```bash
# 在仓库中配置 Git LFS
git lfs install
git lfs track "lalc_backend/deps_macos/*.whl"
git lfs track "lalc_backend/deps_macos/*.tar.gz"
git add .gitattributes
```

**解决方案 B**: 使用 GitHub Releases 附件

```bash
# 将依赖包单独打包
cd lalc_backend
tar -czf deps_macos.tar.gz deps_macos/

# 上传到 GitHub Release 作为附件
# 用户下载后解压到 lalc_backend/ 目录
```
